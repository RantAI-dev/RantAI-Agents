# Workflows

Workflows let you build visual automation pipelines using a drag-and-drop canvas. Chain together AI tasks, integrations, human approvals, and more.

<Screenshot src="/images/workflows/workflow-list.png" alt="Workflow List" />

## What is a Workflow?

A workflow is a sequence of connected **nodes** that execute automatically. You design the flow visually on a canvas, connect nodes with edges, and run it manually, on a schedule, or via webhooks.

## Workflow Modes

| Mode | Description |
|---|---|
| **Standard** | Batch processing — runs start to finish without user interaction mid-flow |
| **Chatflow** | Conversational — supports streaming responses, multi-turn chat, and follow-up suggestions |

## Workflow Categories

| Category | Use For |
|---|---|
| **Task** | One-off or recurring automation tasks |
| **Chatflow** | Conversational AI pipelines |
| **Automation** | Background processes and integrations |

## Workflow Statuses

| Status | Meaning |
|---|---|
| **Draft** | Work in progress, not yet activated |
| **Active** | Ready to run |
| **Paused** | Temporarily disabled |
| **Archived** | No longer in use, hidden from default view |

## Workflow List

The workflows page shows all your workflows with:
- **Status and category badges** — Visual indicators
- **Run count** — How many times the workflow has been executed
- **Last modified** — Relative timestamp
- **Tags** — Color-coded organizational tags
- **Quick actions** — Copy, Download (export), Archive, Delete

Use the filters at the top to narrow by status, category, or tags, and sort by last modified, created date, name, or most runs.
